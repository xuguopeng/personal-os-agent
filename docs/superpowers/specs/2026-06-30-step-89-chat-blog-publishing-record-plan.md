# Step 89：聊天记录博客发布结果

## 目标

让 `@博客` 聊天指令在包含“记录发布 / 已发布 / 已发到公众号草稿箱 / 发布失败”等意图时，优先为已有博客草稿创建一条本地发布记录，并写入流程日志。

## 小步骤

1. 在 Agent 任务结果里增加发布记录结果字段。
2. 增加发布记录意图识别：状态、URL、备注、目标渠道。
3. 选择目标草稿：优先当前聚焦草稿，其次最近更新且未归档的博客草稿。
4. 命中记录意图且存在目标草稿时，创建 `publishing_record`，必要时同步草稿状态。
5. 在聊天回复里说明已记录发布结果；底部日志展示记录动作。
6. 跑前端构建、Rust check/test，并用浏览器输入 `@博客 已发到公众号草稿箱，记录发布` 验证。

## 范围

- 只记录本地发布历史，不自动发布到外部平台。
- 如果消息里没有 URL，记录为待补充的本地占位链接，方便后续编辑补齐。
- 如果找不到已有草稿，不创建发布记录，继续走普通博客草稿生成。

## 验收

- 对已有博客草稿输入发布记录指令后，发布记录列表新增记录。
- 底部流程日志出现 `publishing_record` 和 `agent.blog.record.create`。
- 聊天回复说明记录状态和目标草稿。
- 构建、Rust 检查和测试通过。
