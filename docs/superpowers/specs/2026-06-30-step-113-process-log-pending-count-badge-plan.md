# Step 113：流程日志标题栏显示待确认数量

## 目标

让底部流程日志标题栏直接显示当前任务的真实待确认数量。这样日志折叠、半高或没有滚到待确认卡片时，用户也能知道还有几个动作等待确认。

## 小计划

1. 复用现有 `isActionablePendingConfirmationStep` 过滤规则。
2. 在 App 顶层根据 `taskSteps` 计算真实待确认数量。
3. 在“流程日志”标题栏展示：
   - 总步骤数。
   - 待确认数量，仅在数量大于 0 时显示。
4. 普通通用确认门和 skipped 步骤不计入待确认数量。

## 验证

- `CI=true pnpm build`
- `cargo check`
- `cargo test`
- 浏览器验证：
  - 普通博客草稿任务没有待确认徽标。
  - 删除草稿计划出现 `待确认 1`。
  - 取消删除后待确认徽标消失。
  - console error 为空。
