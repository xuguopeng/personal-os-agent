# Step 49: 发布草稿筛选统计

## 目标

让发布草稿多起来后更容易管理。第一版在前端增加状态统计和状态筛选，不改数据库和后端命令。

## 小步骤

1. 发布草稿面板增加状态筛选：
   - 全部状态
   - 草稿
   - 待发布
   - 已发布
   - 已归档
2. 顶部增加统计条：
   - 总数
   - 草稿
   - 待发布
   - 已发布
   - 已归档
3. 搜索、渠道筛选、状态筛选叠加生效。
4. 空态文案更新为包含状态筛选。
5. 验证：
   - `CI=true pnpm build`
   - `cargo check`
   - `cargo test`
   - 浏览器检查：统计和状态筛选可见，无 console error。

## 非目标

- 不改数据库。
- 不新增后端命令。
- 不真实发布外部平台。
- 不删除草稿。
