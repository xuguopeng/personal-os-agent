# Step 17: 聊天上下文可视化

## 目标

让 Agent 每次回复时，不只输出结果，还能在聊天气泡里清楚看到本次用了哪些长期记忆、哪些知识库资料，以及真实模型调用是否成功。

## 小步骤

1. 扩展前端聊天消息模型：
   - 增加 `contextSummary`，包含记忆命中、知识命中、模型状态和任务 ID。
   - 历史消息如果没有摘要，继续正常显示。
2. 保存策略：
   - 第一版不新增 SQLite 表。
   - 将上下文摘要编码到 assistant 消息的 `modelName` 中。
   - 读取历史消息时自动解析。
3. 聊天 UI：
   - assistant 气泡下方显示“上下文”区域。
   - 展示使用的记忆、知识、模型状态。
   - 点击任务 ID 可在底部日志中查看流程，后续再做直接跳转。
4. 兼容策略：
   - 旧消息只显示模型名。
   - 真实模型返回内容时也保留上下文摘要。
5. 验证：
   - `CI=true pnpm build`
   - `cargo check`
   - `cargo test`
   - 浏览器检查：发送 `@博客` 消息后，回复气泡显示上下文摘要，刷新后仍可恢复。

## 非目标

- 不新增聊天上下文审计表。
- 不做向量命中解释详情。
- 不改变模型调用请求内容。
