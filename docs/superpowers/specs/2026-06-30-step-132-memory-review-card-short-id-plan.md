# Step 132：记忆待确认卡片优先使用短 ID

## 目标

让新创建的记忆候选确认门记录候选短 ID，并让底部“待确认操作”卡片优先给出 `@记忆 批准候选 #短ID` / `@记忆 拒绝候选 #短ID`。短 ID 比内容片段更稳定，候选内容相似时更不容易误操作。

## 小步骤

1. 创建记忆候选确认门时，在 `outputSummary` 写入候选短 ID。
2. 待确认卡片解析 `outputSummary` 中的 `#短ID`。
3. 如果有短 ID，优先生成短 ID 审核命令。
4. 如果没有短 ID，保留内容片段命令作为旧数据兜底。
5. 构建、Rust 检查、浏览器验证。

## 验证

- `CI=true pnpm build`
- `cargo check`
- `cargo test`
- 浏览器验证：
  - 创建一条新的记忆候选。
  - 底部待确认卡片出现 `@记忆 批准候选 #短ID`。
  - 点击“填入”能把短 ID 命令放入聊天输入框。
  - 页面控制台没有应用错误。
