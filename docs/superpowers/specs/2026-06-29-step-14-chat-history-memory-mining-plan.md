# Step 14: 聊天历史搜索与记忆候选提取

## 目标

让长期记忆可以从历史聊天中补充，而不是只能依赖当前一轮对话。记忆中心需要能搜索历史聊天，并从匹配内容中批量生成待确认记忆候选。

## 小步骤

1. 后端新增 `search_chat_messages`：
   - 按关键词搜索 `chat_messages.content`。
   - 默认只搜索用户消息，避免把 Agent 的长回复误当成用户偏好。
   - 返回消息、所属会话标题、创建时间。
2. 前端 backend 封装：
   - Tauri 环境调用真实 SQLite 搜索。
   - 浏览器预览使用 `localStorage` fallback。
3. 记忆中心新增“聊天历史提取”区域：
   - 输入关键词搜索历史聊天。
   - 展示最近匹配消息。
   - 一键从结果中提取记忆候选。
4. 提取规则：
   - 复用当前聊天里的隐式记忆识别规则。
   - 只创建 `pending` 候选。
   - 前端按内容去重，避免同一条候选重复创建。
5. 验证：
   - `CI=true pnpm build`
   - `cargo check`
   - `cargo test`
   - 浏览器检查：搜索历史聊天、提取候选、候选列表刷新。

## 非目标

- 不自动批准长期记忆。
- 不做向量语义搜索，本步先做关键词搜索。
- 不删除历史聊天。
