# Step 39: 任务日志关键节点筛选

## 目标

让底部任务日志在步骤很多时更容易查。第一版只做当前任务内的前端筛选，不新增后端查询和数据库字段。

## 小步骤

1. `ProcessLog` 增加本地步骤筛选状态。
2. 支持筛选：
   - 全部
   - 关键节点
   - 错误
   - 执行相关
   - MCP
   - Skill
   - 模块数据
3. 调用链总览和步骤分组都基于筛选后的步骤展示。
4. 筛选后为空时显示清晰空态。
5. 顶部显示当前匹配步骤数量。
6. 验证：
   - `CI=true pnpm build`
   - `cargo check`
   - `cargo test`
   - 浏览器检查：筛选控件可见，流程日志正常，无 console error。

## 非目标

- 不改数据库。
- 不新增后端命令。
- 不改变真实步骤顺序。
- 不改变任务历史筛选。
