# Step 75: 发布草稿发布前检查

## 目标

在发布草稿卡片上显示发布前准备度，帮助判断一篇草稿是否已经适合进入网站、公众号或自定义渠道发布流程。

## 小步骤

1. 增加 `getPublishingDraftReadiness`。
2. 检查项包括标题、正文长度、具体发布渠道、状态。
3. 卡片标题区域展示 `发布检查 x/4`。
4. 展开草稿时显示每个检查项的通过/待补状态。
5. 检查结果只用于提示，不阻止编辑、下载、记录发布。
6. 不写入数据库，不写入流程日志。
7. 验证：`CI=true pnpm build`、`cargo check`、`cargo test`、浏览器检查博客页可见发布检查且无 console error。

## 非目标

- 不真实发布外部平台。
- 不调用 AI 或 MCP。
- 不修改草稿数据结构。
