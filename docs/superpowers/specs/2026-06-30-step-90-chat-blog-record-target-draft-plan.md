# Step 90：聊天发布记录指定目标草稿

## 目标

让 `@博客` 的“记录发布”指令可以通过标题指定目标草稿，例如 `记录《标题》已发布`，避免只使用当前聚焦草稿或最近草稿。

## 小步骤

1. 增加从聊天内容提取草稿标题线索的函数，支持 `《标题》`、`"标题"`、`“标题”`。
2. 调整发布记录目标草稿选择逻辑：标题匹配优先，其次当前聚焦草稿，再其次最近草稿。
3. 发布记录日志里补充目标草稿选择原因，方便底部流程日志追踪。
4. 跑前端构建、Rust check/test，并用浏览器验证指定标题能命中目标草稿。

## 范围

- 只影响聊天创建本地发布记录的目标草稿选择。
- 不做全文模糊语义检索，第一版先用标题包含匹配。
- 不自动发布外部平台。

## 验收

- 输入 `@博客 记录《某草稿标题》已发布 https://example.com/a` 后，发布记录绑定到标题匹配的草稿。
- 流程日志说明匹配原因。
- 构建、Rust 检查和测试通过。
