# Step 20: 任务历史搜索和筛选

## 目标

当流程日志任务变多后，底部“最近任务”列表需要支持按关键词、模块、状态筛选，方便回看某个功能模块或某次聊天触发的任务。

## 小步骤

1. 后端扩展 `list_task_sessions`：
   - 支持 `query`、`module`、`status`、`limit`。
   - 关键词匹配任务标题、模块、状态。
   - 默认仍返回最近 30 条。
2. 前端 backend 封装：
   - 增加 `TaskSessionFilters`。
   - Tauri 调用真实命令。
   - 浏览器 fallback 本地过滤。
3. App 状态：
   - 增加任务筛选状态。
   - 聚焦任务后保留当前筛选。
4. 底部流程日志 UI：
   - 最近任务列表顶部加入搜索框。
   - 加入模块、状态筛选。
   - 无匹配时显示空态。
5. 验证：
   - `CI=true pnpm build`
   - `cargo check`
   - `cargo test`
   - 浏览器检查：按关键词、模块、状态筛选任务列表，并能点击切换日志。

## 非目标

- 不删除任务。
- 不编辑任务标题。
- 不做分页和无限滚动。
