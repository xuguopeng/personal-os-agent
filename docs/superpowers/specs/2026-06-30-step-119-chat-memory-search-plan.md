# Step 119：聊天查询长期记忆

## 目标

让 `@记忆` 进入聊天驱动工作流。第一版先支持查询长期记忆：用户输入 `@记忆 搜索 ...`、`@记忆 查一下 ...` 时，Agent 会检索已启用的长期记忆，把命中结果返回到聊天里，并写入底部流程日志。

## 小步骤

1. 新增记忆查询意图识别：
   - 识别 `@记忆`、`@长期记忆`。
   - 识别 `搜索`、`查`、`找`、`列出`、`看看`、`有没有` 等查询词。
   - 避开 `记住...` 这类创建候选记忆的语义。
2. 聊天执行时创建 `memory` 任务 session。
3. 使用现有 `retrieveAgentMemories` 检索记忆。
4. 写入流程日志：
   - `intent`
   - `memory_retrieval`
5. 聊天回复直接显示命中数量和前几条记忆。
6. 构建、Rust 检查、浏览器验证。

## 验证

- `CI=true pnpm build`
- `cargo check`
- `cargo test`
- 浏览器验证：
  - 输入 `@记忆 搜索 我喜欢什么` 能得到记忆查询回复。
  - 底部流程日志出现 `memory.retrieve_agent_memories`。
  - 页面控制台没有应用错误。
