# Step 77: 发布渠道配置检查

## 目标

发布渠道不仅要能保存，还要能看出是否已经具备后续发布所需的基础配置。本步在渠道卡片上显示配置检查结果。

## 小步骤

1. 增加 `getPublishingChannelReadiness`。
2. 检查项包括渠道名称、endpoint、认证方式、密钥状态、启用状态。
3. 发布渠道卡片展示 `配置检查 x/5`。
4. 展开编辑渠道时显示每个检查项的通过/待补状态。
5. 检查结果只做提示，不阻止保存、启停或删除。
6. 不回显密钥，不写入流程日志。
7. 验证：`CI=true pnpm build`、`cargo check`、`cargo test`、浏览器检查设置页可见配置检查且无 console error。

## 非目标

- 不真实连接外部网站或公众号。
- 不校验 endpoint 是否可达。
- 不修改数据库结构。
