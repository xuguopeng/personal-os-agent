# Step 38: 任务日志执行节点高亮

## 目标

让底部流程日志里的执行关键节点更容易扫读。已有的执行计划、队列状态变更、执行结果不再混在普通步骤里，而是以更明显的卡片样式展示。

## 小步骤

1. 标记关键执行步骤：
   - `execution_plan`
   - `execution_queue_status`
   - `execution_result`
2. 关键步骤卡片增加视觉高亮：
   - 左侧强调边框
   - 顶部“关键节点”标签
   - 状态色块
3. 执行结果失败时显示更明确的错误区域。
4. 增加执行步骤文案：
   - 队列状态
   - 执行结果
5. 调用链分组继续沿用现有分类，不新增数据结构。
6. 验证：
   - `CI=true pnpm build`
   - `cargo check`
   - `cargo test`
   - 浏览器检查：流程日志和设置页正常，无 console error。

## 非目标

- 不新增数据库字段。
- 不改变步骤写入逻辑。
- 不改变执行队列状态。
- 不接真实 MCP / Skill 执行器。
