# Step 126：记忆确认门显示待确认操作卡片

## 目标

让通过聊天创建的记忆候选，在底部流程日志的“待确认操作”卡片里也能直接操作。卡片提供批准和拒绝两条聊天命令，用户可以一键填入或复制，不需要再手动打开候选列表。

## 小步骤

1. 给 `local.memory_review_gate` 增加建议操作：
   - `@记忆 批准候选《内容片段》`
   - `@记忆 拒绝候选《内容片段》`
2. 更新待确认区域说明文案，覆盖记忆候选、删除、发布、付费生成、导出等场景。
3. 顶部“待确认 N”徽标点击后筛选所有 confirmation 待确认，不再只筛博客删除。
4. 构建、Rust 检查、浏览器验证。

## 验证

- `CI=true pnpm build`
- `cargo check`
- `cargo test`
- 浏览器验证：
  - 创建一条记忆候选。
  - 底部待确认卡片出现批准/拒绝记忆命令。
  - 点击“填入”能把命令放入聊天输入框。
  - 页面控制台没有应用错误。
