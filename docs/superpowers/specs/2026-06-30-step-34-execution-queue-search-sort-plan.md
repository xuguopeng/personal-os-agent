# Step 34: 执行队列搜索和排序

## 目标

让执行队列在任务数量变多后更容易查找。第一版只做前端本地搜索和排序，不新增后端命令，不改变数据库结构。

## 小步骤

1. `ExecutionQueuePanel` 增加本地搜索状态：
   - 搜索标题
   - 搜索模块名称
   - 搜索来源
   - 搜索执行计划 JSON
   - 搜索关联任务 ID
2. 增加排序控件：
   - 最近更新优先
   - 最早更新优先
   - 最近创建优先
   - 状态优先
   - 模块优先
3. 将搜索、状态筛选、模块筛选、dry-run 筛选统一到同一个过滤流程。
4. 列表顶部显示当前匹配数量。
5. 过滤后为空时区分“无队列项”和“无匹配结果”。
6. 验证：
   - `CI=true pnpm build`
   - `cargo check`
   - `cargo test`
   - 浏览器检查：搜索、排序、筛选都可用，且无 console error。

## 非目标

- 不新增后端分页。
- 不新增数据库索引。
- 不执行真实 MCP / Skill。
- 不改变队列写入逻辑。
