# Step 128：内容片段命中多条时给出精确命令

## 目标

当 `@记忆 批准候选《内容片段》` 命中多条 pending 候选时，不只提示“命中多条”，还列出命中的候选和更精确的审核命令。用户可以直接复制其中一条命令继续操作。

## 小步骤

1. 让候选选择结果在 ambiguous 状态下返回匹配候选。
2. 生成最多 5 条候选建议。
3. 在聊天错误回复和流程日志中显示：
   - 候选内容片段
   - `@记忆 批准候选《更完整内容》`
   - `@记忆 拒绝候选《更完整内容》`
4. 构建、Rust 检查、浏览器验证。

## 验证

- `CI=true pnpm build`
- `cargo check`
- `cargo test`
- 浏览器验证：
  - 创建两条包含同一片段的记忆候选。
  - 输入 `@记忆 批准候选《共同片段》`。
  - 聊天回复出现更精确的批准/拒绝命令。
  - 页面控制台没有应用错误。
