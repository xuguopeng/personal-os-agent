# Step 124：按内容审核记忆候选

## 目标

让聊天审核记忆候选不只依赖序号。用户可以输入 `@记忆 批准候选《内容片段》` 或 `@记忆 拒绝候选《内容片段》`，Agent 会优先按内容片段匹配 pending 候选，降低候选列表多时误操作的风险。

## 小步骤

1. 扩展候选审核意图：
   - 保留现有 `@记忆 批准候选 1`。
   - 新增 `《...》`、`“...”`、`"..."`、`'...'` 内容片段提取。
2. 审核时优先使用内容片段匹配 pending 候选。
3. 找不到内容片段时，再按序号匹配。
4. 找不到候选时，在流程日志里说明是内容片段还是序号未命中。
5. 构建、Rust 检查、浏览器验证。

## 验证

- `CI=true pnpm build`
- `cargo check`
- `cargo test`
- 浏览器验证：
  - 创建一条带唯一内容的记忆候选。
  - 输入 `@记忆 批准候选《唯一内容》`。
  - 聊天回复显示已批准。
  - 日志显示 `memory.approve_candidate` 和关闭旧确认门数量。
  - 页面控制台没有应用错误。
