# Step 21: 任务详情与状态管理

## 目标

让底部流程日志显示当前选中任务的完整概要，并支持手动调整任务状态。这样任务日志不只是步骤列表，也能作为工作流任务记录使用。

## 小步骤

1. 后端新增 `update_task_session`：
   - 支持更新任务标题和状态。
   - 状态第一版允许 `draft`、`completed`、`pending`、`error`。
   - 更新后刷新 `updated_at`。
2. 前端 backend 封装：
   - 新增 `updateTaskSession`。
   - 浏览器预览使用 `localStorage` fallback。
3. App 状态：
   - 维护当前选中任务详情。
   - 切换任务、创建任务、更新任务后同步任务列表。
4. 底部流程日志 UI：
   - 右侧顶部显示任务标题、模块、状态、创建时间。
   - 提供状态下拉，可手动标记任务状态。
5. 验证：
   - `CI=true pnpm build`
   - `cargo check`
   - `cargo test`
   - 浏览器检查：切换任务、修改状态、任务列表同步更新。

## 非目标

- 不删除任务。
- 不编辑任务标题。
- 不触发真实执行、发布、导出或付费模型调用。
