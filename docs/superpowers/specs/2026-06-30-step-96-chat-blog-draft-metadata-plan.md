# Step 96：聊天修改博客草稿分类和标签

## 目标

让 `@博客` 聊天指令可以修改已有博客草稿的发布元数据，例如 `把《标题》分类改为随笔，标签改为AI,创作`。

## 小步骤

1. 在 Agent 任务结果里增加草稿元数据更新结果字段。
2. 增加分类/标签更新意图识别，支持只改分类、只改标签或同时修改。
3. 复用标题线索匹配逻辑选择目标草稿。
4. 增加 frontmatter 合并 helper：保留已有元数据，只覆盖 `category` 和 `tags`。
5. 命中元数据更新意图且存在目标草稿时，调用现有草稿更新 API。
6. 追加 `publishing_draft_metadata` 流程步骤，记录旧值、新值和选择原因。
7. 在聊天回复里说明已更新分类/标签。
8. 跑前端构建、Rust check/test，并用浏览器验证指定标题更新元数据。

## 范围

- 只更新本地草稿 frontmatter 的 `category` 和 `tags`。
- 不修改正文主体、状态、渠道或发布记录。
- 不自动发布外部平台。
- 找不到目标草稿时不伪造更新，回退到普通博客草稿创建流程。

## 验收

- 输入 `@博客 把《某草稿标题》分类改为随笔，标签改为AI,创作` 后，目标草稿元数据更新。
- 底部流程日志出现 `publishing_draft_metadata` 和 `agent.blog.draft.metadata`。
- 聊天回复说明分类/标签更新结果。
- 构建、Rust 检查和测试通过。
