# Step 28: 聊天确认卡片

## 目标

把能力策略产生的确认门从“流程日志文字提示”升级为聊天里的可操作确认卡片。用户可以在右侧聊天中直接选择允许执行、拒绝执行或仅生成草稿。

## 小步骤

1. 扩展聊天上下文摘要：
   - 保存是否需要确认。
   - 保存需要确认的能力名称。
   - 保存任务模块和任务 ID。
2. 聊天 UI：
   - assistant 消息下方显示确认卡片。
   - 三个按钮：允许执行、拒绝、仅生成草稿。
   - 已点击后在当前前端状态里显示选择结果。
3. 流程日志：
   - 点击确认按钮后追加 `confirmation_decision` 步骤。
   - 刷新并聚焦对应任务日志。
4. 安全边界：
   - 本步不执行真实 MCP / Skill。
   - “允许执行”只代表用户通过确认门，后续真实执行器再单独实现。
5. 验证：
   - `CI=true pnpm build`
   - `cargo check`
   - `cargo test`
   - 浏览器检查：高风险 Skill 命中后出现确认卡片，点击按钮后底部日志出现确认结果，无 console error。

## 非目标

- 不执行真实外部命令。
- 不发布、导出、剪辑或付费生成。
- 不新增确认历史表；第一版使用 task step 记录。
