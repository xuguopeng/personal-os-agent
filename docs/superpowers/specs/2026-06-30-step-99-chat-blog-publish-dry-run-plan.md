# Step 99：聊天为已有博客草稿创建发布 dry-run 队列

## 目标

让 `@博客` 聊天指令可以为已有博客草稿创建本地发布 dry-run 队列，例如 `准备发布《标题》到公众号`、`把《标题》发布到个人网站`。

## 小步骤

1. 在 Agent 任务结果里增加发布 dry-run 队列结果字段。
2. 增加发布准备意图识别：必须包含标题线索或当前草稿，并表达准备发布/发布到/发到。
3. 复用标题线索匹配逻辑选择目标草稿。
4. 命中发布准备意图且存在目标草稿时，创建 `ExecutionQueueItem`，`dryRun=true`。
5. 追加 `publishing_publish_plan` 和 `execution_plan` 流程步骤，记录目标草稿、目标渠道和队列项 ID。
6. 在聊天回复里说明已生成待执行计划，但不会真实发布。
7. 跑前端构建、Rust check/test，并用浏览器验证队列和日志。

## 范围

- 只创建本地 dry-run 队列项。
- 不调用外部平台，不发送公众号，不发网站。
- 不修改草稿正文。
- 找不到目标草稿时不伪造发布计划，回退到普通博客草稿创建流程。

## 验收

- 输入 `@博客 准备发布《某草稿标题》到公众号` 后，底部流程日志出现 `publishing_publish_plan` 和 `execution_plan`。
- 执行队列新增待执行项。
- 聊天回复说明这是 dry-run，不会真实发布。
- 构建、Rust 检查和测试通过。
