# Step 100：聊天取消博客发布 dry-run 队列

## 目标

让 `@博客` 聊天指令可以取消已有博客草稿的发布 dry-run 队列，例如 `取消《标题》的发布计划`、`取消《标题》的发布队列`。

## 小步骤

1. 在 Agent 任务结果里增加取消发布计划结果字段。
2. 增加取消发布计划意图识别，要求包含标题线索或当前草稿，并表达取消发布计划/发布队列。
3. 复用标题线索匹配逻辑选择目标草稿。
4. 从执行队列中查找对应的 `chat_blog_publish_plan` dry-run 项。
5. 命中队列项时，将状态更新为 `cancelled`。
6. 追加 `publishing_publish_cancel` 和 `execution_queue_status` 流程步骤。
7. 在聊天回复里说明已取消，不影响草稿本身。
8. 跑前端构建、Rust check/test，并用浏览器验证先创建发布 dry-run，再取消。

## 范围

- 只取消本地 dry-run 队列项。
- 不删除草稿，不删除发布记录。
- 不触发外部平台。
- 找不到队列项时只记录未找到，不伪造取消成功。

## 验收

- 输入 `@博客 取消《某草稿标题》的发布计划` 后，对应执行队列项变成已取消。
- 底部流程日志出现 `publishing_publish_cancel` 和 `execution_queue_status`。
- 聊天回复说明队列项已取消。
- 构建、Rust 检查和测试通过。
