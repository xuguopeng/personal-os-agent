# Step 95：聊天修改博客草稿发布渠道

## 目标

让 `@博客` 聊天指令可以修改已有博客草稿的发布渠道类型，例如 `把《标题》发布渠道改为公众号`、`把《标题》切到个人网站`。

## 小步骤

1. 在 Agent 任务结果里增加草稿渠道更新结果字段。
2. 增加渠道更新意图识别，要求同时包含明确修改动词和目标渠道类型。
3. 复用标题线索匹配逻辑选择目标草稿。
4. 命中渠道更新意图且存在目标草稿时，调用现有草稿更新 API，更新渠道类型；如果匹配到具体发布渠道，同步刷新 frontmatter。
5. 追加 `publishing_draft_channel` 流程步骤，记录旧渠道、新渠道和选择原因。
6. 在聊天回复里说明已修改渠道。
7. 跑前端构建、Rust check/test，并用浏览器验证指定标题修改渠道。

## 范围

- 只更新本地草稿渠道类型和可推导的发布目标 frontmatter。
- 不修改正文主体、状态或发布记录。
- 不自动发布外部平台。
- 找不到目标草稿时不伪造更新，回退到普通博客草稿创建流程。

## 验收

- 输入 `@博客 把《某草稿标题》发布渠道改为公众号` 后，目标草稿渠道更新。
- 底部流程日志出现 `publishing_draft_channel` 和 `agent.blog.draft.channel`。
- 聊天回复说明旧渠道和新渠道。
- 构建、Rust 检查和测试通过。
