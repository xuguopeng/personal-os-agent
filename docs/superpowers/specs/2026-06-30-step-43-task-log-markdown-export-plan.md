# Step 43: 任务日志 Markdown 导出

## 目标

把当前任务日志导出成 Markdown，方便复盘 Agent 做了什么，也可以作为博客、项目日志或调试记录素材。

## 小步骤

1. 当前任务头部增加“导出 Markdown”按钮。
2. 导出内容包含：
   - 任务标题、模块、状态、创建时间、更新时间
   - 关联执行队列
   - 执行计划 JSON
   - 全部已加载流程步骤
   - 输入摘要、输出摘要、错误信息、工具名、时间
3. 导出文件名使用任务标题和短 ID。
4. 使用浏览器本地 Blob 下载，不新增后端命令。
5. 验证：
   - `CI=true pnpm build`
   - `cargo check`
   - `cargo test`
   - 浏览器检查：导出按钮可见，流程日志正常，无 console error。

## 非目标

- 不新增数据库字段。
- 不新增后端文件写入。
- 不导出密钥、token、cookies 或 secret。
- 不做 PDF / Word 导出。
