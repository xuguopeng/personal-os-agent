# Step 40: 任务日志步骤搜索

## 目标

让底部任务日志支持按内容定位具体步骤。步骤搜索和第 39 步的类型筛选叠加生效，方便在长流程里查工具调用、输入、输出、错误和执行节点。

## 小步骤

1. `ProcessLog` 增加本地步骤搜索状态。
2. 搜索范围包含：
   - 步骤类型
   - 模块
   - 工具名
   - 输入摘要
   - 输出摘要
   - 错误信息
   - 状态
3. 搜索和现有步骤筛选组合使用。
4. 调用链总览显示当前匹配数量。
5. 筛选或搜索后为空时显示清晰空态。
6. 验证：
   - `CI=true pnpm build`
   - `cargo check`
   - `cargo test`
   - 浏览器检查：步骤搜索框可见，流程日志正常，无 console error。

## 非目标

- 不新增后端搜索。
- 不改数据库。
- 不改变步骤顺序。
- 不做命中词高亮。
