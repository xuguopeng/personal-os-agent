# Step 97：聊天查看博客草稿信息

## 目标

让 `@博客` 聊天指令可以查看已有博客草稿的信息，例如 `查看《标题》状态`、`检查《标题》信息`、`看看《标题》的发布准备情况`。

## 小步骤

1. 在 Agent 任务结果里增加草稿查看结果字段。
2. 增加查看意图识别，支持状态、信息、详情、准备情况、发布准备。
3. 复用标题线索匹配逻辑选择目标草稿。
4. 命中查看意图且存在目标草稿时，解析 frontmatter、计算草稿 readiness 和渠道 readiness。
5. 追加 `publishing_draft_inspect` 流程步骤，记录状态、渠道、分类、标签和检查分数。
6. 在聊天回复里说明草稿当前状态和是否可发布。
7. 跑前端构建、Rust check/test，并用浏览器验证指定标题查看信息。

## 范围

- 只读取本地草稿和渠道配置。
- 不修改草稿、发布记录或外部平台。
- 找不到目标草稿时不伪造查看结果，回退到普通博客草稿创建流程。

## 验收

- 输入 `@博客 查看《某草稿标题》状态` 后，不新建草稿，而是写入查看日志。
- 底部流程日志出现 `publishing_draft_inspect` 和 `agent.blog.draft.inspect`。
- 聊天回复说明状态、渠道和发布检查结果。
- 构建、Rust 检查和测试通过。
