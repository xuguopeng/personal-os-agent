# Step 115：清除流程日志筛选

## 目标

点击标题栏 `待确认 N` 后，流程日志会自动筛选到待确认步骤。本步骤增加一个清除筛选按钮，让用户可以快速回到完整流程日志。

## 小计划

1. 在流程日志搜索框右侧增加清除按钮。
2. 仅当搜索框有内容时显示清除按钮。
3. 点击后：
   - 清空步骤搜索。
   - 筛选恢复为“全部步骤”。
   - 保持当前分组/时间线视图不变。
4. 不改变任务日志、确认门或执行逻辑。

## 验证

- `CI=true pnpm build`
- `cargo check`
- `cargo test`
- 浏览器验证：
  - 点击 `待确认 1` 后搜索框变为 `local.delete_confirmation_gate`。
  - 点击清除按钮后搜索框为空。
  - 调用链总览恢复显示全部步骤。
  - console error 为空。
