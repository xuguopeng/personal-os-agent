# Step 53: 发布草稿归档和删除管理

## 目标

让发布草稿列表具备更完整的本地管理能力：常用草稿可以归档，已归档草稿可以恢复，不需要保留的草稿可以删除。删除动作必须经过确认。

## 小步骤

1. 后端新增 `delete_publishing_draft` 命令。
2. 前端 `backend.ts` 新增 `deletePublishingDraft`，浏览器 fallback 同步支持。
3. 发布草稿列表增加快捷状态动作：
   - 非归档草稿显示“归档”
   - 归档草稿显示“恢复”
4. 发布草稿列表增加“删除”按钮。
5. 删除前使用确认弹窗，确认后才删除数据库记录。
6. 删除或归档后刷新草稿列表，清理展开态和编辑态。
7. 新增 Rust 测试覆盖删除发布草稿。
8. 验证：
   - `CI=true pnpm build`
   - `cargo check`
   - `cargo test`
   - 浏览器检查：博客模块可见归档、恢复、删除入口，无 console error。

## 非目标

- 不做真实外部平台删除。
- 不做回收站。
- 不做批量删除。
- 不做版本历史。
