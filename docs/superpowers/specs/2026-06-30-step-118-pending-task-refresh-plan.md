# Step 118：待确认任务手动刷新

## 目标

给流程日志左侧“只看待确认”增加一个手动刷新入口。新增、确认、取消任务后，用户可以立刻重新扫描最近任务里的待确认状态，不需要切换页面或等待状态自然刷新。

## 小步骤

1. 在 `ProcessLog` 中增加待确认扫描状态：
   - `pendingTaskScanTick`
   - `isPendingTaskScanLoading`
2. 把现有待确认任务扫描逻辑接入刷新触发器。
3. 在“只看待确认”旁增加图标刷新按钮：
   - 只显示图标，不增加长文字。
   - 使用 tooltip 说明功能。
   - 扫描中显示旋转状态。
4. 构建、Rust 检查、浏览器验证。

## 验证

- `CI=true pnpm build`
- `cargo check`
- `cargo test`
- 浏览器验证：
  - 刷新按钮可点击。
  - 点击后待确认数量仍然正确。
  - 页面控制台没有应用错误。
