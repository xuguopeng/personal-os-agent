# Step 105：待确认操作列表

## 目标

把聊天生成的确认门从普通流程日志里提出来，让用户能在底部流程区清楚看到“哪些操作正在等待确认”。第一版只展示，不提供一键执行危险操作。

## 小计划

1. 从当前任务步骤里筛选 `status = pending` 且 `stepType = confirmation` 的步骤，并排除“没有命中外部能力”的通用提示门。
2. 在底部流程日志区域增加“待确认”小节：
   - 显示等待确认数量。
   - 展示模块、工具名、目标摘要、说明。
   - 提醒用户需要通过聊天明确确认，例如 `确认删除《标题》草稿`。
3. 没有待确认项时显示轻量空状态。
4. 保持底部日志可折叠，不改变现有左右工作台/聊天布局。
5. 暂不做一键执行，避免误触真实删除、发布、付费生成。
6. 安全边界：确认词只在标题外生效，标题里包含“确认”等词时，普通删除请求仍然只能生成待确认项。

## 验证

- `CI=true pnpm build`
- `cargo check`
- `cargo test`
- 浏览器验证：
  - 输入普通删除草稿请求。
  - 底部流程区出现待确认项。
  - 标题包含“确认”时，普通删除请求也不能误删。
  - console error 为空。
