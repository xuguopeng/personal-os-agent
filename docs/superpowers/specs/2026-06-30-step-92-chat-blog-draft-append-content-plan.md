# Step 92：聊天追加博客草稿内容

## 目标

让 `@博客` 聊天指令可以给已有博客草稿追加内容，例如 `给《标题》追加一段结尾`、`在《标题》后面补充公众号发布提醒`。

## 小步骤

1. 在 Agent 任务结果里增加草稿内容更新结果字段。
2. 增加追加内容意图识别：追加、补充、加一段、后面加。
3. 复用标题线索匹配逻辑选择目标草稿。
4. 命中追加意图且存在目标草稿时，调用现有草稿更新 API，把内容追加到正文末尾。
5. 追加 `publishing_draft_content` 流程步骤，记录追加字符数和目标草稿。
6. 在聊天回复里说明已追加内容。
7. 跑前端构建、Rust check/test，并用浏览器验证指定标题追加内容。

## 范围

- 只追加本地草稿正文，不覆盖原文。
- 不调用外部模型润色。
- 不自动发布外部平台。
- 找不到目标草稿时不伪造更新，回退到普通博客草稿创建流程。

## 验收

- 输入 `@博客 给《某草稿标题》追加一段：...` 后，目标草稿正文被追加内容。
- 底部流程日志出现 `publishing_draft_content` 和 `agent.blog.draft.append_content`。
- 聊天回复说明追加字符数和目标草稿。
- 构建、Rust 检查和测试通过。
