# Step 87: 聊天创建博客草稿时识别草稿状态

## 目标

`@博客` 已经能根据聊天内容选择发布渠道。本步继续识别草稿状态意图：如果用户说“准备发布”“待发布”“发到公众号”，创建的草稿应进入待发布状态；如果说“先存草稿”，则保持草稿状态。

## 小步骤

1. 增加 `inferBlogDraftStatusFromMessage`。
2. 识别关键词：准备发布、待发布、发布、发到、发公众号、发网站。
3. 识别“先存草稿 / 仅草稿 / 不发布”时保持 `draft`。
4. `@博客` 创建草稿时使用识别出的状态。
5. 流程日志输出状态意图和选择原因。
6. 不真实发布外部平台。
7. 验证：`CI=true pnpm build`、`cargo check`、`cargo test`、浏览器输入 `@博客 ... 准备发布到公众号` 后草稿创建且无 console error。

## 非目标

- 不自动发布到网站或公众号。
- 不自动创建发布记录。
- 不修改发布渠道或草稿数据结构。
