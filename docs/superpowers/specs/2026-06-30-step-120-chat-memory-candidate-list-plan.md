# Step 120：聊天查看待确认记忆候选

## 目标

让 `@记忆` 可以通过聊天查看待确认的记忆候选。用户输入 `@记忆 待确认`、`@记忆 候选列表`、`@记忆 看看待确认记忆` 时，Agent 会列出当前 pending 候选，并把查看过程写入底部流程日志。

## 小步骤

1. 新增记忆候选列表意图识别：
   - 识别 `@记忆`、`@长期记忆`。
   - 识别 `待确认`、`候选`、`候选列表`、`待审核`、`待处理`。
2. 聊天执行时创建 `memory` 任务 session。
3. 使用现有 `listMemoryCandidates("pending")` 读取候选。
4. 写入流程日志：
   - `intent`
   - `memory_candidate`
5. 聊天回复显示候选数量和前几条候选内容。
6. 构建、Rust 检查、浏览器验证。

## 验证

- `CI=true pnpm build`
- `cargo check`
- `cargo test`
- 浏览器验证：
  - 输入 `@记忆 待确认` 能看到候选列表回复。
  - 底部流程日志出现 `memory.list_candidates`。
  - 页面控制台没有应用错误。
