# Step 15: 记忆去重与来源可见性

## 目标

让记忆候选系统在数据变多后仍然可信：候选不能和已有长期记忆重复，也不能和待确认候选重复；用户能在候选列表和历史提取结果里看到来源线索与跳过原因。

## 小步骤

1. 后端增加重复检测辅助测试：
   - 同一内容在 `memory_candidates` 中能被识别为重复。
   - 同一内容在 `memory_items` 中能被识别为已存在长期记忆。
2. 前端历史提取去重增强：
   - 同时读取候选和长期记忆。
   - 对已存在候选、已确认长期记忆分别显示不同原因。
3. 记忆候选来源可见：
   - 候选列表显示 `sourceEventId` 的短 ID。
   - 历史提取结果显示“可提取 / 已在候选 / 已是长期记忆 / 未命中规则”。
4. 流程日志增强：
   - 搜索步骤记录找到数、可提取数、跳过数。
   - 当没有新候选时显示明确跳过原因。
5. 验证：
   - `CI=true pnpm build`
   - `cargo check`
   - `cargo test`
   - 浏览器检查历史提取结果状态和底部日志。

## 非目标

- 不删除重复历史数据。
- 不自动合并长期记忆。
- 不做向量相似去重，本步只做规范化文本去重。
