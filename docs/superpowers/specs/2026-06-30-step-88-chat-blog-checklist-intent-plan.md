# Step 88：聊天触发博客发布清单

## 目标

让 `@博客` 聊天指令在包含“发布清单 / 检查 / 发布前检查”等意图时，不只创建博客草稿，还要把发布清单生成动作写入底部流程日志，并在聊天回复里说明清单已经生成。

## 小步骤

1. 在 Agent 任务结果里增加博客清单生成标记。
2. 增加本地意图识别函数，判断聊天内容是否要求生成发布清单或发布前检查。
3. 在 `@博客` 创建草稿后，复用现有草稿检查、渠道检查和 Markdown 清单格式化逻辑，追加一条 `publishing_checklist` 任务步骤。
4. 在流式聊天回复里补充“已生成发布清单，可在草稿卡片复制/下载”的提示。
5. 跑前端构建、Rust check/test，并用浏览器验证输入 `@博客 ... 发布清单` 后日志和页面不报错。

## 范围

- 只做本地清单生成和流程日志记录。
- 不自动复制剪贴板。
- 不自动发布到网站或公众号。
- 不调用外部付费模型或真实发布 API。

## 验收

- 输入 `@博客 帮我写一篇发布到公众号的草稿，并生成发布清单` 后，博客模块出现新草稿。
- 底部流程日志包含 `publishing_checklist` 步骤。
- 聊天回复说明发布清单已生成。
- 构建、Rust 检查和测试通过。
