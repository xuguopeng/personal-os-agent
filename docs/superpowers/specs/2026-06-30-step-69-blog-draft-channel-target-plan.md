# Step 69: 博客草稿选择具体发布渠道

## 目标

新建博客草稿时可以选择已配置的具体发布渠道，而不是只能选择“个人网站 / 微信公众号 / 自定义渠道”这类渠道类型。草稿保存时带上渠道默认分类和标签，方便后续发布到网站或公众号。

## 小步骤

1. `BlogDraftComposer` 接收发布渠道列表。
2. 草稿目标下拉优先展示已配置渠道，并标记启用/停用状态。
3. 没有合适渠道时保留渠道类型兜底选项。
4. 选择具体渠道后，草稿的 `channelType` 使用该渠道类型。
5. 保存草稿时，如果选择了具体渠道，则给 Markdown 正文补充 frontmatter：渠道名、渠道类型、账号标识、默认分类、默认标签。
6. frontmatter 不写入密钥、token、cookie。
7. 如果正文已经有 frontmatter，则不重复插入。
8. 验证：`CI=true pnpm build`、`cargo check`、`cargo test`、浏览器检查博客新建草稿目标下拉无 console error。

## 非目标

- 不修改草稿数据库结构。
- 不真实发布到网站或公众号。
- 不自动创建发布记录。
