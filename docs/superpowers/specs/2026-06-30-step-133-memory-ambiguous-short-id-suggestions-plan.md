# Step 133：模糊命中建议优先使用短 ID

## 目标

当 `@记忆 批准候选《内容片段》` 命中多条候选时，错误回复和日志恢复按钮优先给出 `@记忆 批准候选 #短ID` / `@记忆 拒绝候选 #短ID`。这样用户从模糊错误继续操作时，也能用最稳定的候选定位方式。

## 小步骤

1. 更新模糊命中摘要，候选建议使用短 ID 命令。
2. 保留候选内容展示，方便用户识别是哪一条。
3. 扩展错误日志恢复命令提取，支持 `#短ID` 和旧的 `《内容片段》` 两种格式。
4. 构建、Rust 检查、浏览器验证。

## 验证

- `CI=true pnpm build`
- `cargo check`
- `cargo test`
- 浏览器验证：
  - 创建两条包含同一片段的记忆候选。
  - 输入 `@记忆 批准候选《共同片段》`。
  - 回复里出现 `@记忆 批准候选 #短ID`。
  - 错误日志恢复按钮能把短 ID 命令填入聊天。
  - 页面控制台没有应用错误。
