# Step 111：待确认卡片提供确认和取消命令

## 目标

让待确认操作卡片不仅提示“确认删除”，也提示“取消删除”。用户可以一键把任一命令填入聊天框，但不会自动发送。

## 小计划

1. 把单条建议命令改为建议动作列表。
2. 删除草稿确认门提供两条动作：
   - 确认：`@博客 确认删除《标题》草稿`
   - 取消：`@博客 取消删除《标题》草稿`
3. 每条动作都支持：
   - 填入聊天
   - 复制命令
4. 复制状态按按钮分别反馈，避免一个按钮影响另一个按钮。
5. 保持真实执行仍由聊天发送触发，不在卡片里直接执行。

## 验证

- `CI=true pnpm build`
- `cargo check`
- `cargo test`
- 浏览器验证：
  - 普通删除草稿后，待确认卡片显示确认/取消两条命令。
  - 点击“填入”确认命令，聊天输入框出现确认删除命令，不自动发送。
  - 点击“填入”取消命令，聊天输入框替换为取消删除命令，不自动发送。
  - console error 为空。
