# Step 13: 聊天会话管理

## 目标

把已经落库的聊天记录从“自动打开最近一条”升级为可使用的会话系统。Agent 右侧聊天需要能新建会话、切换历史会话、显示当前会话标题，并在第一条用户消息后自动生成易读标题。

## 小步骤

1. 后端补齐会话命令：
   - `list_chat_sessions`
   - `create_chat_session`
   - `update_chat_session`
2. 数据策略：
   - `chat_sessions` 继续只存普通元数据。
   - 会话标题默认是“新会话”。
   - 第一条用户消息发送后，如果标题仍是默认标题，则自动改成消息前 18 个字符。
3. 前端 backend 封装：
   - Tauri 环境调用真实命令。
   - 浏览器预览继续使用 `localStorage` fallback。
4. Agent 聊天界面：
   - 头部显示当前会话选择器。
   - 支持点击新建会话。
   - 切换会话后加载对应消息。
   - 当前会话为空时显示默认助手欢迎语，不写入数据库。
5. 验证：
   - `CI=true pnpm build`
   - `cargo check`
   - `cargo test`
   - 浏览器检查：新建会话、发送消息、切换旧会话、刷新后仍能恢复。

## 非目标

- 不做会话删除，避免误删聊天记忆来源。
- 不做真实多模型并发对话。
- 不做聊天全文搜索，后续单独实现。
