# Step 103：聊天生成博客草稿删除确认计划

## 目标

让 `@博客` 聊天指令在用户要求删除草稿时，不直接删除数据，而是生成一个需要确认的删除计划日志，例如 `删除《标题》`、`移除《标题》草稿`。

## 小步骤

1. 在 Agent 任务结果里增加删除确认计划字段。
2. 增加删除意图识别，要求包含标题线索或当前草稿，并表达删除/移除。
3. 复用标题线索匹配逻辑选择目标草稿；明确标题未命中时不回退。
4. 命中删除意图且存在目标草稿时，不调用删除 API，只追加 `publishing_draft_delete_plan` 和 `confirmation` 流程步骤。
5. 在聊天回复里说明已生成删除确认计划，未删除草稿。
6. 跑前端构建、Rust check/test，并用浏览器验证删除请求不会删除草稿。

## 范围

- 不真实删除草稿。
- 不删除发布记录、队列或日志。
- 只写入流程日志和聊天回复。
- 后续确认卡片执行删除另做步骤。

## 验收

- 输入 `@博客 删除《某草稿标题》` 后，草稿仍存在。
- 底部流程日志出现 `publishing_draft_delete_plan` 和 `confirmation`。
- 聊天回复明确说明未删除，只生成确认计划。
- 构建、Rust 检查和测试通过。
