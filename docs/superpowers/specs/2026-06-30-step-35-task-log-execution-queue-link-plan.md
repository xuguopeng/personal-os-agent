# Step 35: 任务日志和执行队列双向关联

## 目标

把底部流程日志和设置页执行队列串起来。用户查看某个任务时，可以直接看到这个任务生成了哪些执行队列项；查看队列项详情时，仍然可以回到对应任务日志。

## 小步骤

1. `ProcessLog` 接收执行队列列表。
2. 根据当前激活的 `taskSessionId` 找出关联队列项。
3. 在任务头部下方展示“关联执行队列”：
   - 队列标题
   - 状态
   - 模块
   - dry-run 标识
   - 更新时间
   - 计划摘要
4. 在任务日志里增加“打开执行队列”入口，跳转到设置页执行队列区域所在模块。
5. 保留执行队列详情里的任务回跳能力，形成双向关联。
6. 验证：
   - `CI=true pnpm build`
   - `cargo check`
   - `cargo test`
   - 浏览器检查：日志区域、设置页执行队列可见，且无 console error。

## 非目标

- 不新增数据库字段。
- 不新增后端查询命令。
- 不做真实 MCP / Skill 执行。
- 不做跨页面滚动定位锚点。
