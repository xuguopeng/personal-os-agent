# Step 16: 记忆候选来源详情

## 目标

让记忆候选不只是显示一个来源短 ID，而是可以打开来源详情，看到它来自哪条聊天或哪次任务，以及相关流程日志。

## 小步骤

1. 后端新增 `get_memory_source_context`：
   - 如果来源 ID 是 `chat_messages.id`，返回聊天消息、会话标题和关联任务日志。
   - 如果来源 ID 是 `task_sessions.id`，返回任务会话和任务步骤。
   - 如果找不到，返回 `unknown` 状态。
2. 前端 backend 封装：
   - Tauri 环境调用真实命令。
   - 浏览器预览从 `localStorage` 里的聊天、任务和日志记录解析。
3. 记忆候选 UI：
   - 来源短 ID 改成可点击按钮。
   - 打开详情面板，显示来源类型、标题、原始聊天内容或任务信息、流程步骤。
4. 交互策略：
   - 不做编辑来源。
   - 不删除来源数据。
   - 找不到来源时展示清楚的说明。
5. 验证：
   - `CI=true pnpm build`
   - `cargo check`
   - `cargo test`
   - 浏览器检查：点击来源能打开详情，聊天来源和任务来源均能显示。

## 非目标

- 不做完整时间线跳转。
- 不做聊天记录删除或任务日志删除。
- 不做来源数据迁移。
