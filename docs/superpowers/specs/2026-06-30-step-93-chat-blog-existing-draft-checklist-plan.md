# Step 93：聊天给已有博客草稿生成发布清单

## 目标

让 `@博客` 聊天指令可以对已有博客草稿生成发布清单，例如 `给《标题》生成发布清单`、`检查《标题》的发布前清单`。

## 小步骤

1. 在 Agent 任务结果里增加已有草稿清单结果字段。
2. 增加“已有草稿清单”意图识别：必须包含发布清单/发布检查，并带有标题线索或“当前草稿”。
3. 复用标题线索匹配逻辑选择目标草稿。
4. 复用现有草稿检查、渠道检查和 Markdown 清单格式化逻辑，追加 `publishing_checklist` 流程步骤。
5. 在聊天回复里说明已给目标草稿生成发布清单。
6. 跑前端构建、Rust check/test，并用浏览器验证指定标题生成清单。

## 范围

- 只生成本地发布检查和日志记录。
- 不复制剪贴板。
- 不自动发布外部平台。
- 如果没有标题线索或当前草稿指向，则继续走原来的新建草稿流程。

## 验收

- 输入 `@博客 给《某草稿标题》生成发布清单` 后，不新建草稿，而是给目标草稿写入清单日志。
- 底部流程日志出现 `publishing_checklist` 和 `agent.blog.checklist.generate_existing`。
- 聊天回复说明目标草稿和清单状态。
- 构建、Rust 检查和测试通过。
