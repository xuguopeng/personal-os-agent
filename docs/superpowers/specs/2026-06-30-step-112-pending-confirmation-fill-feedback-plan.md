# Step 112：待确认卡片填入反馈

## 目标

点击待确认卡片的“填入”按钮后，按钮短暂显示“已填入”，让用户明确知道命令已经写入右侧聊天输入框。该动作仍然不自动发送。

## 小计划

1. 为每条建议动作增加独立填入状态。
2. 点击“填入”后：
   - 调用现有 `onFillChat(command)`。
   - 当前动作按钮显示“已填入”。
   - 约 1.4 秒后恢复“填入”。
3. 确认/取消两条动作互不影响。
4. 保留复制按钮的独立反馈。

## 验证

- `CI=true pnpm build`
- `cargo check`
- `cargo test`
- 浏览器验证：
  - 点击确认动作“填入”后，输入框出现确认删除命令，按钮显示“已填入”。
  - 点击取消动作“填入”后，输入框替换为取消删除命令，按钮显示“已填入”。
  - 不自动发送，console error 为空。
