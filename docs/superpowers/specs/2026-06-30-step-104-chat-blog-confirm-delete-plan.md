# Step 104：聊天确认删除博客草稿

## 目标

把 Step 103 的“删除确认计划”推进到受控执行：只有用户在聊天里明确输入 `确认删除《标题》草稿`、`确定删除《标题》草稿` 或同等明确表达时，才删除本地发布草稿。

## 小计划

1. 新增确认删除意图识别，必须同时满足：
   - 有草稿标题线索或“当前草稿”。
   - 有删除词。
   - 有确认/确定/执行/真的/delete confirmed 等明确确认词。
   - 排除发布计划、发布队列、dry-run 等语义。
2. 聊天分支优先处理确认删除，再处理普通删除计划。
3. 删除成功后写入任务步骤：
   - `publishing_draft_delete_confirm`
   - tool：`agent.blog.draft.delete.confirm`
   - 状态：`success`
   - 输出包含标题、草稿 ID、选择原因、删除前状态。
4. 删除后刷新发布草稿列表，清空当前聚焦草稿。
5. 回复中明确说明“已删除本地草稿”，并提示不会影响外部网站/公众号已发布内容。

## 验证

- `CI=true pnpm build`
- `cargo check`
- `cargo test`
- 浏览器验证：
  - 创建测试草稿。
  - 输入普通删除，确认不会删除。
  - 输入确认删除，确认搜索不再命中该测试草稿。
  - 检查 console error。
