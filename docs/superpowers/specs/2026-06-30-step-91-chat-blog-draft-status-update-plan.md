# Step 91：聊天更新博客草稿状态

## 目标

让 `@博客` 聊天指令可以更新已有博客草稿状态，例如 `把《标题》标记为待发布`、`归档《标题》`、`把《标题》恢复为草稿`。

## 小步骤

1. 在 Agent 任务结果里增加草稿状态更新结果字段。
2. 增加状态更新意图识别：草稿、待发布、已发布、已归档。
3. 复用 Step 90 的标题线索匹配逻辑选择目标草稿。
4. 命中状态更新意图且存在目标草稿时，调用现有草稿状态更新 API，并追加 `publishing_draft_status` 流程步骤。
5. 在聊天回复里说明更新前后状态。
6. 跑前端构建、Rust check/test，并用浏览器验证指定标题更新状态。

## 范围

- 只更新本地草稿状态。
- 不修改正文内容。
- 不自动发布外部平台。
- 找不到目标草稿时不伪造更新，回退到普通博客草稿创建流程。

## 验收

- 输入 `@博客 把《某草稿标题》标记为待发布` 后，目标草稿状态更新。
- 底部流程日志出现 `publishing_draft_status` 和 `agent.blog.draft.status`。
- 聊天回复说明原状态和新状态。
- 构建、Rust 检查和测试通过。
