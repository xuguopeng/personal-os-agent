# Step 94：聊天重命名博客草稿

## 目标

让 `@博客` 聊天指令可以重命名已有博客草稿，例如 `把《旧标题》重命名为《新标题》`、`把《旧标题》标题改为《新标题》`。

## 小步骤

1. 在 Agent 任务结果里增加草稿重命名结果字段。
2. 增加重命名意图识别，优先读取第二个书名号/引号作为新标题。
3. 复用标题线索匹配逻辑选择目标草稿。
4. 命中重命名意图且存在目标草稿时，调用现有草稿更新 API，仅更新标题。
5. 追加 `publishing_draft_rename` 流程步骤，记录旧标题、新标题和选择原因。
6. 在聊天回复里说明已重命名。
7. 跑前端构建、Rust check/test，并用浏览器验证指定标题重命名。

## 范围

- 只更新本地草稿标题。
- 不修改正文、状态或发布记录。
- 不自动发布外部平台。
- 找不到目标草稿时不伪造更新，回退到普通博客草稿创建流程。

## 验收

- 输入 `@博客 把《旧标题》重命名为《新标题》` 后，目标草稿标题更新。
- 底部流程日志出现 `publishing_draft_rename` 和 `agent.blog.draft.rename`。
- 聊天回复说明旧标题和新标题。
- 构建、Rust 检查和测试通过。
