# Step 86: 聊天创建博客草稿时识别目标渠道

## 目标

`@博客` 已经能创建草稿并打开博客模块。本步让 Agent 根据聊天内容选择目标发布渠道，例如提到“公众号”时优先选择微信公众号渠道，提到“网站”时优先选择个人网站渠道。

## 小步骤

1. 增加 `selectBlogDraftChannelForMessage`。
2. 识别关键词：公众号 / 微信 / wechat、网站 / 站点 / website、自定义 / custom。
3. 优先选择匹配类型且已启用的渠道。
4. 如果没有已启用渠道，则选择匹配类型的任意渠道。
5. 如果没有匹配类型，则回退到第一个已启用渠道或第一个渠道。
6. `@博客` 创建草稿时使用该渠道写入 frontmatter。
7. 流程日志输出目标渠道和选择原因。
8. 验证：`CI=true pnpm build`、`cargo check`、`cargo test`、浏览器输入 `@博客 ... 公众号` 后仍能创建草稿且无 console error。

## 非目标

- 不真实发布到公众号或网站。
- 不自动创建发布记录。
- 不修改发布渠道数据结构。
