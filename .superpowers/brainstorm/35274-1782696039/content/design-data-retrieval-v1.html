补充设计：数据分层和检索路径

数据多了以后，Agent 不能把所有内容都读进上下文；它要先过滤，再检索，再控制 token。

Agent Retrieval Pipeline
Knowledge Base
项目文档 / 小说设定 / 博客素材
标签 / 项目 / 模块 / 全文搜索
向量索引：语义查找资料
Memory Base
创作偏好 / 工作习惯 / 娱乐偏好
可查看 / 可编辑 / 可禁用 / 可删除
调用时显示用了哪些记忆
Task Log
任务步骤 / MCP / skill / 模块工具
耗时 / token / 输入输出摘要
默认不进上下文，只按需查最近摘要
意图识别 → 模块/项目/标签过滤 → 全文搜索 → 向量搜索 → Top-K 摘要 → token 预算 → Agent 回复
AI Config
聊天模型：对话、规划、生成草稿
Embedding 模型：生成向量，用于语义搜索

关键结论

向量索引需要配置 embedding AI，但搜索系统不能完全依赖它。没有 embedding 时，标签、项目、模块和全文搜索仍然可用；配置 embedding 后，Agent 才能更聪明地做语义检索。
